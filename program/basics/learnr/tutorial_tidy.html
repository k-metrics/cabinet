About Tutorial

本チュートリアルではBTS（Bug Tracking System）データの集計を事例としてdplyrパッケージ、tidyrパッケージを用いた整然データ（Tidy Data）の扱い方をモダンなRのコーディングで処理するために必要な基礎知識を学びます。なお、本チュートリアルを実行するための必要な環境についてはこちらで確認して下さい。

必要なパッケージ

整然データを処理するためには必須とも言えるdplyrパッケージやtidyrパッケージはtidyverseパッケージに含まれています。tidyverseパッケージとは共通の設計思想に基づき相互の連携が容易なRのパッケージコレクションで、モダンなRのプログラミングでは必須のパッケージです。

必要なデータ

前述のように具体的な事例としてBTSのデータを用います。BTSのデータはRedmine.JPで公開されているサンプルDBを用いますので、予めCSV形式でエクスポートしておいて下さい。

エクスポート

サンプルDBはエクスポートできるチケット数が50に限定されていますので以下の手順で二回のエクスポートを行いファイルをマージして下さい。

  1. フィルタに表示されている［ステータス］のチェックを外す
  2. フィルタ追加のドロップダウンリストから［トラッカー］を選択する
  3. ［トラッカー］にチェックがありドロップダウンリストが［等しい］、［Defects］になっていることを確認し［適用］リンクをクリックする
  4. データが表示されたら画面下部にあるたの形式にエクスポートの［CSV］リンクをクリックする
  5. ［すべての項目］を選択し［説明］にチェックを入れて［エクスポート］ボタンをクリックする
  6. ファイルがダウンロードされたらファイル名を「issues_defect.csv」に変更する
  7. ［トラッカー］にチェックがありドロップリストを［等しい］、［Patch］に変更し［適用］リンクをクリックする
  8. データが表示されたら画面下部にあるたの形式にエクスポートの［CSV］リンクをクリックする
  9. ［すべての項目］を選択し［説明］にチェックを入れて［エクスポート］ボタンをクリックする
  10. ファイルがダウンロードされたらファイル名を「issues_patch.csv」に変更する
  11. 「issues_defect.csv」と「issues_pacth.csv」をマージして「issues.csv」ファイルにしておく

参考までにRでファイルをマージするコード以下に示します。マージ結果はプロジェクトのホームにUTF8形式で保存されます。

  dplyr::bind_rows(patch) %>%
  dplyr::arrange(`#`) %>%
  readr::write_csv("issues.csv")
rm(defect, patch)

Rでのモダンな分析

本チュートリアルではdplyrパッケージならびにtidyrパッケージによるパイプを使ったコーディング（モダンなコーディング）を用います。このコーディングは文字通りパイプ演算子（%>%）を用いたコーディングです。パイプ演算子とは演算子の左側の値（主にデータフレーム型）を右側の第一引数として渡す演算子で、例えばファイルパスにあるファイルを読み込んで表示するような場合、以下のような記述が可能になります。

パイプ演算子による記述

file_path %>%                       # ファイル
  readr::read_csv(locale = locale(encoding = "CP932")) %>% 
                                    # CP932(S-JIS)ととして読み込む場合
  DT::datatable()                   # 読み込み結果を表示

この様にパイプ演算子は中間変数を作成することなく処理の手順を記述できます。コードの可読性が上がるというメリットもあります。パイプ演算子を使わない以下の場合と比較して見て下さい。

中間変数を用いた記述

bugs <- readr::read_csv(file_path, locale = locale(encoding = "CP932"))
DT::datatable(bugs)

関数をネストする記述

# 関数をネストする場合
DT::datatable(
  readr::read_csv(file_path, locale = locale(encoding = "CP932"))
)

Import/Overview

まず、BTSデータをインポートします。インポート方法には複数の方法が考えられますが、本チュートリアルでは環境構築の手間を考慮してファイルからインポートする方法を用います。実際の環境においてはSQLによるアクセスが使える場合には、分析自動化の観点からもSQLによるインポートを実装しておくことをおすゝめします。

ファイルからインポートする

ファイルからデータをインポートする場合、ファイル形式としてはテキスト形式であるCSV形式が最も汎用的です。RでCSVファイルをインポートするには基本パッケージで提供されているutil::read.csv関数を用いるのが一般的に紹介されている方法です。ただ、BTSのデータの様に様々な形式のデータが混在しているファイルを読み込むにはreadr::read_csv関数の方が適しています。

readr::read_csv関数の引数は以下のようになっています。詳細はヘルプで確認して下さい。

read_csv(file, col_names = TRUE, col_types = NULL,
  locale = default_locale(), na = c("", "NA"), quoted_na = TRUE,
  quote = "\"", comment = "", trim_ws = TRUE, skip = 0, n_max = Inf,
  guess_max = min(1000, n_max), progress = show_progress())

注意すべきは以下のような点です。

  • ファイル先頭行に列名（ヘッダ）が含まれているか否か（col_names）
  • 欠損値の扱い方（na）
  • ファイルのエンコーディング（locale）

特に複数のプラットフォームが混在している場合は、ファイルのエンコーディングに注意して下さい。読み込んだファイルが文字化けする場合は以下のようにエンコーディングを明示的に指定して下さい。

locale = locale(encoding = "CP932")     # シフトJISコードの場合
